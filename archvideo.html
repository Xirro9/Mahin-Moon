Creating a Custom Version of Linux using Arch



Documentation




I began by downloading the Arch Linux ISO and configuring the VM, proceeding to manually partition the hard drive and install essential packages via pacstrap, including the base system, kernel, firmware, text editor, networking tools, and GRUB bootloader. After generating the fstab file, setting the root password, and rebooting, I created a new user, installed sudo, updated the system, and set up Xorg and Plasma KDE. Enabling SDDM allowed me to access the graphical login screen. I then created another user, enforced password expiry, and edited the sudoers file directly for sudo privileges. Further customization involved installing zsh, SSH, and adding terminal colors. Finally, I configured the system to boot into the GUI DE by default and added aliases for convenience, completing the setup of my Arch Linux system.
